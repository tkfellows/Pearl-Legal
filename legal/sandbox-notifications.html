1. Overview

Pearl uses Apple's App Store Server Notifications to ensure secure, reliable, and transparent processing of in-app purchases and subscriptions. This system provides real-time updates about transaction status, subscription changes, and billing events.

Purpose: App Store Server Notifications help Pearl maintain accurate billing records, prevent fraud, and provide users with transparent transaction information.

2. What Are App Store Server Notifications?

App Store Server Notifications are secure, encrypted messages sent by Apple to Pearl's servers when specific events occur related to your purchases. These notifications include:

  • Purchase Events: Initial purchases, renewals, and cancellations
  • Subscription Changes: Upgrades, downgrades, and billing updates
  • Refund Processing: Refund requests and processing status
  • Account Changes: Family sharing updates and account transfers
  • Billing Issues: Payment failures and retry attempts

3. Notification Types

Pearl receives and processes the following types of App Store Server Notifications:

CONSUMPTION_REQUEST
Apple requests consumption information for consumable in-app purchases to prevent duplicate purchases.
DID_CHANGE_RENEWAL_PREF
User changed their subscription renewal preference (e.g., from monthly to annual).
DID_CHANGE_RENEWAL_STATUS
User enabled or disabled subscription auto-renewal.
DID_FAIL_TO_RENEW
Subscription renewal failed due to payment issues.
DID_RENEW
Subscription successfully renewed for another period.
EXPIRED
Subscription expired and was not renewed.
GRACE_PERIOD_EXPIRED
Grace period for failed payment has expired.
OFFER_REDEEMED
User redeemed a promotional offer or discount code.
PRICE_INCREASE
Subscription price increased and user was notified.
REFUND
Purchase was refunded by Apple or user.
REFUND_DECLINED
Refund request was declined by Apple.
RENEWAL_EXTENDED
Subscription renewal period was extended.
REVOKE
Purchase was revoked (e.g., due to fraud detection).
SUBSCRIBED
User successfully subscribed to a service.
TEST
Test notification sent by Apple for verification.

4. Security and Privacy

4.1 Encryption and Authentication

  • All notifications are encrypted using industry-standard TLS/SSL protocols
  • Apple verifies notification authenticity using cryptographic signatures
  • Pearl validates each notification before processing
  • Secure webhook endpoints with authentication tokens

4.2 Data Protection

  • Notification data is processed securely and never shared with third parties
  • Personal information is minimized and encrypted at rest
  • Access to notification data is restricted to authorized personnel only
  • Regular security audits and penetration testing

5. Technical Implementation

5.1 Webhook Endpoints

Pearl maintains secure webhook endpoints to receive App Store Server Notifications:

Production: https://api.pearlapp.com/webhooks/appstore
Sandbox: https://api.pearlapp.com/webhooks/appstore/sandbox

5.2 Notification Processing

  • Real-time notification reception and validation
  • Automatic transaction status updates
  • Subscription state synchronization
  • Error handling and retry mechanisms
  • Comprehensive logging for audit trails

6. Data Handling and Storage

6.1 Data Collection

  • Transaction identifiers and purchase tokens
  • Subscription status and renewal information
  • Billing events and payment status
  • App Store account identifiers (anonymized)
  • Notification timestamps and event types

6.2 Data Retention

  • Transaction records retained for 7 years (tax compliance)
  • Subscription data retained while active plus 2 years
  • Notification logs retained for 90 days for debugging
  • Anonymized analytics data retained indefinitely

7. User Controls and Preferences

  • Users can view their purchase history in the App Store
  • Subscription management through Apple ID settings
  • Refund requests processed through Apple Support
  • Data export available upon request
  • Account deletion removes all associated data

8. Compliance and Standards

8.1 App Store Guidelines

  • Full compliance with Apple's App Store Review Guidelines
  • Implementation of required App Store Server Notifications
  • Proper handling of subscription status changes
  • Transparent pricing and billing practices

8.2 Legal Compliance

  • GDPR compliance for European users
  • CCPA compliance for California residents
  • HIPAA compliance for medical data handling
  • PCI DSS compliance for payment processing
